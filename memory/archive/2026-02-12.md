## 记忆机制讨论

### 关于 OpenClaw 的记忆保存机制

用户询问了记忆系统如何自动识别和保存关键信息。

**机制说明：**
- 记忆不是"自动触发"的，而是基于 AGENTS.md 中的规则
- 我需要主动识别对话中的重要信息并写入文件
- 规则来源：AGENTS.md 中的 "Write It Down" 部分

**关键规则：**
1. 有人明确说"记住这个" → 更新 memory/YYYY-MM-DD.md
2. 学到教训 → 更新 AGENTS.md、TOOLS.md 或相关技能
3. 犯错 → 记录避免重复
4. 核心原则：Text > Brain（文件比大脑可靠）

**触发时机：**
- 对话中主动识别重要信息
- Heartbeat 定期维护（审查近期 memory 文件，更新 MEMORY.md）
- 每次会话开始时读取记忆文件

**文件结构：**
```
memory/
├── 2026-02-12.md    # 当天原始记录
├── 2026-02-11.md    # 昨天
└── MEMORY.md        # 精选长期记忆
```

## 创建 Spark - 内容创意写作 Agent

今天创建了 Spark，一个文艺型的内容创意写手。

### 用户需求
- 写作方向：公众号文章 + X（Twitter）平台帖子
- 使用场景：独立助手，随时调用
- 核心能力：搜索资料、了解用户写作风格
- 名字：Spark（火花）
- 风格：文艺型

### Spark 的特点

**写作理念：**
- 少用华丽，多用真心
- 让文字呼吸
- 故事胜过说教
- 简洁不是简陋

**写作原则：**
- 开篇 3 秒抓人，但不要标题党
- 结构清晰，起承转合自然
- 有观点但不偏激
- 结尾留白

**平台适配：**
- 公众号：深度思考，800-2000 字
- X：短而有力，< 280 字，有互动性

### 文件结构

```
spark/
├── SOUL.md       # Spark 的性格和风格
├── AGENTS.md     # 写作规则和工作流
├── USER.md       # 用户信息（待补充）
├── memory/
│   └── 2026-02-12.md   # 诞生记录
└── README.md     # 使用说明
```
