# 2026-02-17 调试日志：Discord 响应与 Gateway 授权修复

## 事件背景
在 Discord 服务器 `openbot` 的 `#日常` 频道中，用户希望实现无需 @ 也能自动回复的功能。在调试过程中出现了 Gateway 授权失效以及配置字段错误的问题。

## 关键诊断
1. **授权冲突**：运行 `openclaw gateway status` 发现 `device token mismatch` 错误。这导致 Agent 虽然能读取消息，但无法通过 WebSocket 发送回复。
2. **配置字段错误**：误用了不存在的 `autoReplyChannels` 字段，导致配置更新失败。
3. **响应逻辑约束**：`requireMention: true` 的全局设置限制了非提及状态下的回复。

## 解决方案
1. **刷新授权**：通过 `openclaw gateway restart` 重启服务，重新对齐 CLI 与 Gateway 的 Token。
2. **优化配置**：在 `openclaw.json` 中，将该 Guild 的 `requireMention` 设置为 `false`。
3. **流程规范**：确立了“修改配置必先提供 Diff、必先查阅 Schema、必经用户确认”的原则。

## 结果
- `#日常`、`#开发`、`#创作`、`#简报` 四个频道现在均支持不带 @ 的直接对话。
- 授权恢复正常，Agent 回复功能完全恢复。

## 经验总结
- **验证第一**：修改 `openclaw.json` 前必须对比官方文档/Schema。
- **日志优先**：遇到“已读不回”优先检查 `gateway status`。
- **手动介入**：当 `gateway` 工具因 Token 错误失效时，直接通过 `edit` 文件并调用 `openclaw gateway restart` 命令更可靠。
