# 实战教程：解决 OpenClaw “已读不回”的 Token 冲突

**状态**：已完稿
**风格版本**：v1.1 (Writer Skill - Zack Style)
**日期**：2026-02-20

---

😇最近在折腾 OpenClaw 自动回复的时候，遇到了一个挺头疼的问题。明明后台能看到消息，但 Agent 就是不回话，简直像个“哑巴”狗子。

排查了一圈才发现，原来是 Gateway 的授权令牌（Token）没对上。这时候哪怕你改再多配置都没用，最简单的办法就是去终端跑一下 `openclaw gateway restart`，让系统强制同步一下“钥匙”。

还有一个容易被忽略的点是 `requireMention` 这个设置。默认情况下，你必须在频道里 @ 它才会回话。如果你想让它像真人一样随时能插嘴聊天，记得去 `openclaw.json` 里把它关掉。

最后想说，玩这种 Agent 最重要的就是别怕报错。每解决一个像 Token 冲突这样的小坑，其实都是在跟你的 AI 建立更好的默契。只要基础环境稳了，剩下的就全是乐趣了😀
